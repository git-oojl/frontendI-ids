Conceptos clave

1) Programación Orientada a Objetos (POO) y clases anidadas

  • Tienda funciona como “contenedor” de empleados y define los métodos para administrar la información.
  • Empleado es una clase relacionada dentro de Tienda (clase anidada) para modelar cada registro.
class Tienda {
  static Empleado = class Empleado {
    constructor({ nombre, puesto, salario, identificador }) { /* ... */ }
  };
}

2) Manejo de eventos

  • Usamos addEventListener para reaccionar al envío del formulario y a los botones (mostrar, filtrar, ordenar).
  • Para “Eliminar”, se usa delegación de eventos en el <tbody> (más eficiente que un listener por fila).
form.addEventListener("submit", (e) => {
  e.preventDefault();
  // agregar empleado...
});

3) Manipulación del DOM (render dinámico)

  • La tabla se construye dinámicamente con createElement y appendChild.
  • Si no hay empleados, se inserta una fila con el mensaje “Sin empleados para mostrar”.
const tr = document.createElement("tr");
const td = document.createElement("td");
td.textContent = empleado.nombre;
tr.appendChild(td);
tbody.appendChild(tr);

4) Almacenamiento local (localStorage)

  • Guardamos el arreglo en localStorage con JSON.stringify para conservarlo al recargar.
  • Al iniciar, cargamos con JSON.parse y rehidratamos la información en instancias de Empleado.
// Guardar
localStorage.setItem("tienda_empleados_v1", JSON.stringify(empleados));

// Cargar
const empleados = JSON.parse(localStorage.getItem("tienda_empleados_v1") || "[]");

5) Separación de responsabilidades (lógica vs interfaz)

  • Model: clases Tienda / Empleado (reglas del negocio).
  • Storage: guardar/cargar en localStorage.
  • UI: lectura de inputs, render de tabla y mensajes (DOM).
